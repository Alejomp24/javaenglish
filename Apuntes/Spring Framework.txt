ITERACIÓN 1: TIGHT COUPLING

El acoplamiento es una medida del trabajo que supone cambiar algo.
Acoplamiento estrecho se refiere al minimo trabajo para cambiar algo.

Por ejemplo en el proyecto de ejecución de videojuegos, tenemos que estar cambiando constantemente el código para que se ejecuté un videojuego.

Preguntas para definir el Acoplamiento:

¿Cuánto trabajo implica cambiar algo?

- El acoplamiento es una medida del trabajo que supone cambiar algo.
- Acoplamiento estrecho se refiere al minimo trabajo para cambiar algo.
- Por ejemplo en el proyecto de ejecución de videojuegos, la clase GameRunner está estrechamente vinculada a un juego específico.
	Por lo cual tenemos que estar cambiando constantemente el código para que se ejecuté otro.
- Queremos poder realizar cambios funcionales con el menor número posible de cambios de código.

ITERACIÓN 4: AUTO WIRING

Se puede personalizar el nombre del bean, por ejemplo @Bean(name = "address2")

Otra forma de obtener el bean sería: System.out.println(context.getBean(Person.class));

El Spring permite definir múltiples beans y permite una amplia gama para recuperar los objetos que están siendo gestionados.

Hasta el momento todos los Beans que se han creado son indpendientes:

public String name()
public int age()
public Person person()
public Address address()

Cómo hacer para integrarlos:

	@Bean
	public Person person2MethodCall() {
		return new Person(name(), age(), address());
	}


PREGUNTAS:

1.¿Spring Container vs Spring Context vs IOC Container vs Application Context?

Spring Container: Administra Spring beans y su ciclo de vida.
	
	1.Bean Factory: Basic Spring Container.
	
	2.Application Context: Advance Spring Container con características específicas de la empresa:
		*Fácil de usar en las aplicaciones Web.
		*Fácil de internacionalización.
		*Fácil integración con Spring AOP (Orientes Programming)
		*Recomendado paea Web Services, Web Applications, REST API y Microservices

2.¿Java Bean vs Spring Bean?

Java Bean: 
*Es antiguo, 
*Es poco usado por la parte empresarial, 
*Requiere mayor implementación de código (definir la clase como Serializable, se debe crear un constructor vacio y se deben crear setter & getters para los atributos).

Spring Bean:
Así que cualquier cosa que sea gestionada por spring framework es lo que se llama un spring bean.
Utiliza todo lo de la pregunta 1.

3.¿Cómo puedo listar todos los beans gestionados por spring framework?

Con los siguientes metodos:
	.getBeanDefinition
	.getBeanDefinitionCount
	.getBeanDefinitionNames
	
	Ejemplo: Arrays.stream(context.getBeanDefinitionNames()).forEach(System.out::println);



4.¿Qué pasa si hay varios beans coincidentes disponibles?

Este es el caso que tiene la instrucción NombreBean.class y el ejecutar el software falla, porque él no sabe a qué Bean debe dirigirse.
	Ejemplo: Address.class

Para solucionarlo uno de los beans debe ser primario @Primary y también se puede solucionar usando una calificación @Qualifier
